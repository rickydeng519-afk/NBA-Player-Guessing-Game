# CLAUDE.md — NBA 猜球员项目工作指引

## 项目简介

NBA 球员猜测游戏（Poeltl 风格）。每日一个谜底球员，用户 8 次猜测机会，通过 7 个属性（Team/Conf/Div/Pos/Ht/Age/#）比对来寻找答案。

## 关键文件路径

| 文件 | 路径 | 说明 |
|------|------|------|
| 开发需求 | `docs/requirements.md` | 功能与非功能需求 |
| 技术规范 | `docs/tech-spec.md` | 数据结构、算法、API |
| 设计规范 | `docs/design-spec.md` | 色彩、排版、布局、动效 |
| 执行步骤 | `docs/execution-plan.md` | 分阶段实施清单 |
| 开发日志 | `devlog/YYYY-MM-DD.md` | 每日工作记录 |

## 每个开发会话的工作流程

1. **查看执行计划**：读取 `docs/execution-plan.md`，了解当前阶段和待办项
2. **查看设计规范**：写 UI 代码前查阅 `docs/design-spec.md` 的色彩和布局规范
3. **记录日志**：会话结束后在 `devlog/` 中更新当日日志
4. **更新执行计划**：标记已完成的任务

## 核心规则（不可偏离）

1. **零依赖**：所有代码在单个 `index.html` 中，不引入任何外部库或框架
2. **数据内嵌**：球员数据作为 JS 数组直接写在 HTML 中
3. **移动优先**：所有样式以 375px 视口为基础向上适配
4. **色板固定**：严格使用 `design-spec.md` 中定义的颜色值
5. **每日刷新**：谜底由日期哈希决定，不可随机或手动指定

## 编码约定

- HTML：语义化标签，`<table>` 用于猜测结果
- CSS：CSS 变量集中定义，BEM 风格命名（如 `.guess-row__cell--green`）
- JS：ES6+，函数式为主，避免深层嵌套
- 注释：关键算法加注释，复杂逻辑写说明
- 数据：`const PLAYERS` 全大写，不可修改
